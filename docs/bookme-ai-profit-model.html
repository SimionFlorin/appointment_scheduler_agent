BookMe AI — Revenue → Cost → Profit Schema

Distilled from cursor_temporary_docs/app-overview.md §§3–5. Only sources that are contractually committed are included — speculative future revenue (§3.3) is excluded. Nothing here is in production with paying Tenants today; the schema describes the model the codebase is being built towards.

Absolute net profit, per Period (currently 1 calendar month) 
For one billing Period:

  Revenue           =  for every Tenant, take ( R1 + R2 − R3 ),  then add up all Tenants
  Variable cost     =  for every Tenant, take ( C1 + C2 + C3 + C4 ),  then add up all Tenants
  Fixed cost        =  C5 + C6 + C7/12 + C8 + C9
  Acquisition cost  =  C10 + C11 + C12 + C13 + C14
  One-off cost      =  C15 + C16 + C17 + C18 + C19
  Labour cost       =  C20 + C21 + C22


  NET PROFIT  =   Revenue
              −   Variable cost
              −   Fixed cost
              −   Acquisition cost
              −   One-off cost
              −   Labour cost

Pass-through Costs P1 (WhatsApp messaging) and P2 (Tenant's Google Workspace) are deliberately absent — they are paid by the Tenant directly and never hit BookMe AI's books under today's credential-paste model (see §4.7 / §9). C7 is divided by 12 because the domain is billed annually but Periods are monthly.

Diagram

Each cost group flows into the same destination — the Period's net profit. Revenue is added (green); costs are subtracted (neutral).

Schema at a glance

Revenue (R1 · R2 · R3) +

R1 Subscription Fee per Tier per Period (includes Token Allowance).

R2 Token Overage = provider unit price + Markup (cost-plus).

R3 Promotional discount adjustments (negative revenue).

Costs (C1 – C22) −

Variable C1–C4 — scale with Tenants & traffic.

Fixed C5–C9 — paid at zero Tenants.

Acquisition C10–C14 — bounded by our decision.

One-off C15–C19 — milestone / legal.

Labour C20–C22 — paid only.

Profit (three views) =

Per-Tenant gross margin — Tenant's revenue minus the variable costs that Tenant caused.

Platform contribution margin — total revenue minus all variable & fixed costs.

Net profit — contribution margin minus acquisition, one-off and labour.

3 · Three profit views (§5.1)

Per-Tenant gross margin
For ONE Tenant, in one Period:

GROSS MARGIN  =   ( R1 + R2 − R3 )
              −   ( C1 + C2 + C3 + C4 )

R2 is structurally cost-plus, so every Token above the Allowance contributes exactly the Markup.

Platform contribution margin
For ALL Tenants combined, in one Period:

CONTRIBUTION  =   total Revenue from Tenants
              −   total Variable cost (across Tenants)
              −   ( C5 + C6 + C7/12 + C8 + C9 )

Per-Tenant margin can be positive while contribution margin is still negative, because fixed costs are paid at zero Tenants.

Net profit
For the whole platform, in one Period:

NET PROFIT  =   Contribution margin
            −   ( C10 + C11 + C12 + C13 + C14 )    acquisition
            −   ( C15 + C16 + C17 + C18 + C19 )    one-off (period share)
            −   ( C20 + C21 + C22 )                labour

Steerable: pause C10–C11 and net profit rises without changing the product.
